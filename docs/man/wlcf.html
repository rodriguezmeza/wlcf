NAME

       wlcf - Weak Lensing Correlation Functions computation code

SYNOPSIS

       wlcf [ parameter_file_name ] [ options ]

DESCRIPTION

       wlcf computes weak lensing correlation functions using perturbation
       theory, effective field theory, and halo-model inspired methods.

       The code is designed to compute multipoles of the three-point
       correlation function (3PCF) of the weak lensing convergence field in a
       plane-wave expansion.

OPTIONS

       All command-line options have the structure

       $ option_name=<option_value>

       except the option --help. No spaces should be used between option_name,
       '=' and option_value in command-line mode.



       --help      Prints the list of available parameters and their default
                   values:

                   $ wlcf --help


                   Some options may have aliases. In the help output, aliases
                   are indicated with the tag [a: alias]. For example:

                   option_name=<option_value>     Description     [a: opt]



       paramfile   Name of the file containing input parameter values.
                   Parameters in this file overwrite default values.

                   $ wlcf paramfile=parameters_input_file


                   Parameter files can be created manually. Comment lines
                   start with "#" or "%".  In a parameter file, use the
                   format:

                   option_name = option_value


                   The order of parameters does not matter. Running the code
                   with default values:

                   $ wlcf

                   creates a file with the parameters used in the run, for
                   example:

                   parameters_null-wlcf-usedvalues

                   or a similarly named file inside the output directory. This
                   file can be used as a template for custom parameter files.

COSMOLOGICAL PARAMETERS

       z           Redshift used in the computation.


       h           Dimensionless Hubble parameter.


       sigma8      [a: s8] Normalization of the matter power spectrum.


       Omb         Baryon density parameter.


       Omc         Cold dark matter density parameter.


       Omnu        Massive neutrino density parameter.


       ns          Spectral index of the linear power spectrum.


       w           Dark energy equation of state parameter.

INPUT PARAMETERS

       fnamePS     [a: ps] Linear matter power spectrum file.


       Wg          Weak lensing kernel option. If Wg=0, the code uses a Dirac
                   delta at the redshift bin. If Wg=1, the kernel is read from
                   file.


       fWgchi      [a: fwgchi] File containing the weak lensing kernel
                   Wg(chi).

MODEL AND NUMERICAL PARAMETERS

       tree_level  Selects the theoretical model used for the bispectrum
                   calculation:

                   1 = Standard Perturbation Theory (SPT)
                   2 = P2 approximation
                   3 = Effective Field Theory (EFT)
                   4 or larger = Takahashi/Halo-model inspired bispectrum


       zbin        Redshift bin used in the projection.


       mMax        Maximum multipole order to compute.


       chiQuadSteps
                   [a: chiqst] Number of integration steps in comoving
                   distance.


       GLpoints    [a: gl] Number of Gauss-Legendre integration points.


       Nell        Number of angular multipole values.


       ellmin      Minimum multipole value.


       ellmax      Maximum multipole value.

OUTPUT PARAMETERS

       rootDir     [a: root] Output directory where files will be written. If
                   the directory does not exist, it will be created.


       prefix      [a: pre] Prefix used for output file names.


       path_Bells  [a: bellout] Directory where angular correlation function
                   outputs are written.


       writevectors
                   If true, intermediate vectors are written to output files.

MISCELLANEOUS PARAMETERS

       verbose     [a: verb] Controls the amount of information printed to
                   standard output.


       verbose_log [a: verblog] Controls the amount of information written to
                   the log file.


       chatty      Additional verbosity level for internal messages.


       numberThreads
                   [a: nthreads] Number of OpenMP threads used during
                   execution. OpenMP must be enabled at compilation time.


       options     [a: opt] Additional code behavior options.

EXAMPLES

       Compile the executable:

       $ make clean; make


       Compile the executable and Python wrapper:

       $ make clean; make all


       Run the default test:

       $ cd tests
       $ ../wlcf


       Run with a custom linear power spectrum:

       $ ../wlcf ps=./input/linear_pk_Takahashi_z0.txt verb=2


       Run the EFT branch:

       $ ../wlcf tree_level=3 ps=./input/linear_pk_Takahashi_z0.txt


       Run the Takahashi/Halo-model branch:

       $ ../wlcf tree_level=4 ps=./input/linear_pk_Takahashi_z0.txt

PYTHON INTERFACE

       The Python wrapper can be used as:

       $ python

       >>> from wlcfpy import wlcf
       >>> w = wlcf()
       >>> w.set(numberThreads=8, tree_level=4)
       >>> w.Run()

OUTPUT FILES

       The code writes output files using the selected prefix. Typical output
       files include:

       theta_array.txt
       karray.txt
       Bmells_*.txt
       Bnk_*.txt
       zetam*.txt


       The files zetam*.txt contain the multipoles of the 3PCF, zeta_m(theta1,
       theta2).
